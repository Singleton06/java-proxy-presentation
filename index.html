What is a Proxy Class?

A dynamic proxy class (simply referred to as a proxy class) is a class that implements a list of interfaces specified at runtime when the class is created.

Other Proxy Details

  • Each proxy has an invocation handler
  • All proxy classes will extend java.lang.reflect.Proxy
  • instanceof checks will work
  • casting to interfaces the proxy implements will work

Invocation Handler

Each proxy class will have an invocation handler. Basically, this is just an object that indicates how method calls on the proxy class should be handled.

Invocation Handler Cont

java.lang.reflect.InvocationHandler is an interface that has one method called 'invoke' which is called every time that a method is called on the proxy object. This is the interface you will generally implement.

Invoke Arguments

Implementing the invoke method gives you this data to work with:

  • Object proxy - the instance of the proxy that was called
  • java.lang.reflect.Method method - the method that was called
  • Object[] args - the arguments that were supplied to the method call
